## Setup
OK SO
Open up your favorite sus editor and open susidian_stuffs
Then put in the CD your obsidian vault's directory, surrounded by double quotes ("smthortheother/Obsidian Vault")
In the URL, put the HTTPS URL generated by your favorite geethub :000
Then put these in a place that you can run them easily (or find a way to add them to path or smth idk maybe ill figure out how to do that when its not actively 5 am)
ok so its now SIX am so that means its ok :D
If you choose, you can also use the BAT file, which you can then add to PATH, allowing easy "susidian -e"s and other fun schmorgus borgus
In order to use the bat(man), find the path to your Git file and then insert it in the spot in susidian.bat(man). Then add the file (batman file I mean) to your (pedro)path variables.

## Running 
not from the demons
Running this is as simple as running this.
just run the shell (if you open powershell on windows, you can just find the directory of the folder the files are in, then find folder relative to the current user, then append on the end the file name (if I had "C:/Users/Jimothy/Documents/Secret/Not Launch Codes" as my path, and I was just running it from the standard powershell, then I would use "Documents/Secret/Not Launch Codes/susidian.sh")) and then append at the end one of 3 flags.

-i: allows you to "implode" or pull info from github into obsidian
-e: allows you to "explode" or push info to geethub from obsidian, with the default commit message of "UPDATED :D \[Insert (and torke) Year/Month/Day or maybe Day/Month/Year lowk im too tired for this\]"
-m: basically -e, but you input your own message after the m with double quotes (-m "he no like the banana") (appended by the daymonthyearthingy)

if you used the batman method, you can simply run it from cmd, using "susidian -\[thingymabop\]" to do things :D
